# Security Policy: Cripta Identity Framework (CIF)

## 🛡️ Our Philosophy
CIF is built on the "Path of Least Resistance" but with "Zero Compromise" on integrity. We believe in transparency regarding our cryptographic choices.

<br />

## ⚠️ Disclosure Policy
**Do not report security vulnerabilities via public GitHub issues.**

If you discover a potential security flaw, please help us protect the community by reaching out via the secure contact methods listed on my GitHub profile:
👉 **[Social Links](https://github.com/harnuma9)**

We aim to acknowledge all valid reports within 48 hours.

<br />

## 📦 Supported Versions
| Version | Supported          |
| ------- | ------------------ |
| v1.x    | ✅ Active (Stable)  |
| < v1.0  | ❌ Not Supported    |

<br />

## 🔍 Audit Status

CIF is currently in an **Experimental Stable** phase. 

- **External Audit:** Pending funding/collaboration.
- **Internal Audit:** Continuous. The core engine is self-verified via ML-DSA signatures.

<br />

## 🚫 Zero-Access Guarantee

The author and the framework itself have **zero access** to your Master CID, PIN, or derived secrets. All entropy remains local to your environment.
